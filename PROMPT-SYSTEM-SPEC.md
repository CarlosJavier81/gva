# Sistema de Prompts Jurídicos por Proceso (Workflow-Driven)

## 1. Propósito del sistema

Este sistema tiene como objetivo **ordenar y estandarizar la forma en que los abogados interactúan con modelos de inteligencia artificial**, mediante plantillas de prompts estructuradas según el proceso jurídico.

El sistema:
- NO reemplaza el criterio jurídico
- NO toma decisiones legales
- NO genera documentos finales
- SÍ organiza instrucciones
- SÍ reduce errores y riesgos
- SÍ estandariza calidad

---

## 2. Principio rector

> El sistema no “piensa” ni razona jurídicamente.  
> El sistema **ensambla instrucciones jurídicas** definidas previamente.

Toda generación de prompts es:
- determinística
- auditable
- controlada

---

## 3. Estructura base de un Prompt Legal

Todo prompt del sistema debe componerse **obligatoriamente** de los siguientes bloques:

1. **Tarea**
2. **Contexto fáctico**
3. **Contexto jurídico**

Ningún prompt puede omitir estos tres elementos.

---

## 4. Organización por Proceso Jurídico

El sistema organiza los prompts según las siguientes **etapas del proceso judicial**:

1. Prelitigio  
2. Marco Jurídico  
3. Jurisprudencia  
4. Estrategia Legal  
5. Redacción  
6. Revisión  
7. Control Final  

Cada etapa tiene:
- un objetivo específico
- una plantilla de prompt propia
- un nivel de riesgo distinto

---

## 5. Niveles de profundidad del Prompt

El sistema soporta tres niveles de complejidad:

### Nivel A – Prompt Legal Mínimo
- Solo tarea + contexto fáctico + contexto jurídico
- Uso inicial
- Riesgo bajo

### Nivel B – Prompt Legal Medio
- Incluye sub-instrucciones
- Mayor especificidad
- Uso frecuente

### Nivel C – Prompt Legal Avanzado
- Análisis estructurado
- Simulación de perspectivas
- Control de calidad
- Uso por abogados con experiencia

Todos los niveles mantienen la **misma estructura procesal**, solo cambia la profundidad.

---

## 6. Funcionamiento del sistema

El sistema funciona de la siguiente manera:

1. El usuario selecciona una **etapa del proceso**
2. El usuario selecciona un **nivel de profundidad (A, B o C)**
3. El sistema carga una **plantilla fija** correspondiente
4. El usuario completa los campos variables (hechos, normas)
5. El sistema **ensambla el prompt** sin interpretarlo
6. El prompt se entrega como texto copiable

El sistema no modifica ni evalúa el contenido ingresado por el usuario.

---

## 7. Qué el sistema NO hace

El sistema NO:
- interpreta hechos
- valida normas
- accede a bases legales
- evalúa calidad jurídica
- decide estrategias
- genera documentos finales
- sustituye la revisión humana

---

## 8. Responsabilidad profesional

Todo uso del sistema requiere revisión humana.

El resultado de la IA:
- es apoyo
- no es asesoría legal autónoma
- no exime responsabilidad profesional

---

## 9. Ejemplo conceptual de ensamblaje

Estructura de ensamblaje:

Tarea:
[texto fijo por etapa y nivel]

Contexto fáctico:
[input del usuario]

Contexto jurídico:
[texto fijo por etapa y nivel]
[input opcional del usuario]

---

## 10. Objetivo final del sistema

El objetivo final es crear una **biblioteca institucional de prompts jurídicos**, coherente, escalable y segura, que acompañe al abogado en su trabajo sin reemplazarlo.
